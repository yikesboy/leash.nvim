INTRODUCTION                                          *leash.nvim-introduction*

leash.nvim is a review-first orchestration layer for coding-agent CLIs. The
plugin is designed to run an agent in an isolated workspace, stream progress,
and let the user review proposed file changes before applying them to the real
workspace.

This initial package provides the plugin skeleton, configuration, and command
registration. Agent execution and review UI behavior are implemented by later
work packages.

==============================================================================
SETUP                                                        *leash.nvim-setup*

Minimal setup:
>
  require("leash").setup()
<

With selected options:
>
  require("leash").setup({
    adapter = "codex",
    ui = {
      file_list_width = 32,
      log_height = 12,
    },
  })
<

==============================================================================
COMMANDS                                                  *leash.nvim-commands*

                                                        *:LeashStart*
:LeashStart [adapter] {prompt}
        Start a new agent session. The current implementation creates and
        persists session metadata; agent execution is implemented by later
        packages.

                                                        *:LeashOpen*
:LeashOpen
        Open the current review.

                                                        *:LeashFiles*
:LeashFiles
        Focus the changed-files list.

                                                        *:LeashAcceptFile*
:LeashAcceptFile
        Accept all changes for the current file.

                                                        *:LeashRejectFile*
:LeashRejectFile
        Reject all changes for the current file.

                                                        *:LeashAbort*
:LeashAbort
        Abort the current agent run.

                                                        *:LeashResume*
:LeashResume [session_id]
        Load a persisted session and make it current. Without an ID, the most
        recently updated persisted session is used.

                                                        *:LeashPrompt*
:'<,'>LeashPrompt
        Prompt using the current visual selection as context.

==============================================================================
CONFIGURATION                                      *leash.nvim-configuration*

Defaults:
>
  {
    adapter = "codex",
    adapters = {
      codex = {
        command = "codex",
        extra_args = {},
        sandbox = "workspace-write",
        approval_policy = nil,
      },
      claude = {
        command = "claude",
        extra_args = {},
        permission_mode = nil,
        allowed_tools = nil,
      },
    },
    review = {
      use_worktree = true,
      accept_hunks = false,
      auto_open = true,
      diffopt = "filler,context:3,linematch:60,inline:word",
    },
    limits = {
      max_file_size = 1024 * 1024,
      max_file_lines = 20000,
    },
    ui = {
      file_list_width = 32,
      log_height = 12,
      keymaps = true,
    },
    persistence = {
      enabled = true,
      cleanup_worktrees = "manual",
    },
    safety = {
      allow_unsafe_agent_options = false,
    },
  }
<

Unsafe agent options are rejected by default when passed through adapter
extra_args. Users must explicitly set:
>
  safety = {
    allow_unsafe_agent_options = true,
  }
<

==============================================================================
PERSISTENCE                                            *leash.nvim-persistence*

Session metadata is stored under:
>
  stdpath("state")/leash/sessions/{session_id}/
<

The current package writes:
>
  session.json
  decisions.json
  events.jsonl
  summary.json
<

Session metadata intentionally excludes raw process environments and large file
contents. Later packages add agent execution, changed-file discovery, and review
decisions on top of this storage layout.

==============================================================================
STATUS                                                      *leash.nvim-status*

The current implementation includes the skeleton package, session state,
persistence, the adapter registry, the streaming job runner, and a deterministic
fake adapter for tests. Real Codex and Claude adapters, worktree capture, diff
review, and accept/reject behavior are implemented by later packages.
